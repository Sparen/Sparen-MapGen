Usage Instructions

MapGen is a tool to assist in the creation of transit maps. It uses JSON to define various fields.

It is imperative that proper JSON formatting is used - invalid or malformed JSON will result in errors when the system tries to render the map.

Getting Started

If you are a first time user, we suggest loading a pre-made map as a reference (see the buttons above). You will notice lines prefixed by // - these are comments and will be ignored when a map is generated. These comments only work when the // are the first two characters in a line..

Hit the Generate Map button, and the JSON in the text box will be parsed to generate a SVG map in the empty box. There are also buttons for Save and Load - this will be discussed later.

Global Values

Within every map, there are a number of values at the top that are required for all maps.

svgwidth: Width of the SVG Canvas, in pixels

svgheight: Height of the SVG Canvas, in pixels

globaloffset: Offset used for shifting contents of map, in pixels. First value corresponds to x direction, second value corresponds to y direction. Y direction increases doing downwards. Use [0, 0] if you do not want an offset.

bgcolor: Color of the SVG Canvas. Supports Hex, rgb, and HTML colors.

stationlabelfont: Font used for all station labels

stationlabelfontsize: Font size used for all station labels
